# Progress Log Guide

Reference example:
(see existing entries)

Use this when:
- You did work
- You changed layout or routing
- You discovered an issue

Tags:
- [WIP] → ongoing work
- [OPEN] → unresolved issue
- [DECISION] → major choice made
- [CONSTRAINT] → external limitation
- [TEST NEEDED] → unverified assumption

Rules:
- Keep entries short
- Always include why it mattered (1 line)
- Do NOT explain deeply (that belongs in decision files)

Structure:
- Bullet = action
- Sub-bullet = meaning

Daily Summary:
- Direction → what improved
- Risk → biggest concern
- Next Focus → next likely step
